3.1 Overview

In the digital age, vast amounts of data are available online. Collecting and utilizing this data effectively is crucial for data science and AI applications. This module focuses on:

  • Web Scraping: Extracting data from websites using tools like rvest in R.
  • APIs: Accessing structured data through Application Programming Interfaces.
  • Ethical Considerations: Understanding the legal and ethical implications of data collection.
  • AI Summarization: Using AI models, such as GPT, to summarize and interpret collected data.

3.2 Learning Objectives

By the end of this module, learners will be able to:

1.  Understand the differences between web scraping and API-based data collection.
2.  Use R and the rvest package to scrape data from web pages.
3.  Identify and adhere to ethical guidelines in data collection.
4.  Utilize AI tools to summarize and interpret collected data.

3.2.1 Section 1: Web Scraping with R

What is Web Scraping?

Web scraping involves programmatically extracting data from websites. It’s useful when data isn’t available through APIs or downloadable formats.

Tool: rvest Package in R

The rvest package simplifies web scraping by providing functions to:

  • Read HTML content.
  • Navigate and extract elements using CSS selectors.
  • Convert extracted data into usable formats like data frames.

Example: Scraping a Wikipedia Table

Let’s scrape a table from Wikipedia, such as the list of countries by GDP.

Code 
# Load necessary libraries
library(rvest)
library(dplyr)

# Define the URL
url <- "https://en.wikipedia.org/wiki/List_of_countries_by_GDP_(nominal)"

# Read the HTML content
page <- read_html(url)

# Extract the first table on the page
gdp_table <- page %>%
  html_node("table.wikitable") %>%
  janitor::clean_names() %>%
  html_table()

# View the first few rows
head(gdp_table)

Explanation:

  • read_html() reads the HTML content of the page.
  • html_node(“table.wikitable”) selects the first table with the class wikitable.
  • html_table() converts the HTML table into a data frame.

However, it returns bad data because of the first two rows. We can use some tricks to clean the names and remove the first two rows, thanks to ChatGPT.

Code 
# Load necessary libraries
library(rvest)
library(dplyr)
library(janitor)
library(stringr)

# Define the URL
url <- "https://en.wikipedia.org/wiki/List_of_countries_by_GDP_(nominal)"

# Read the HTML content
page <- read_html(url)

# Extract all tables with the class 'wikitable'
tables <- page %>% html_nodes("table.wikitable")

# Select the first table
gdp_table <- tables[[1]] %>% html_table(header = FALSE)

# Combine the first two rows to create column names
combined_headers <- paste(gdp_table[1, ], gdp_table[2, ], sep = " ")
combined_headers <- str_remove_all(combined_headers, "\\[.*?\\]")
combined_headers <- str_trim(combined_headers)
colnames(gdp_table) <- combined_headers

# Remove the first two rows from the data
gdp_table <- gdp_table[-c(1, 2), ]

# Clean the column names
gdp_table <- gdp_table %>% janitor::clean_names()
names(gdp_table)[names(gdp_table) == "country_territory_country_territory"] <- "country_territory"

# View the cleaned data
head(gdp_table)

3.2.2 Section 2: Ethical Considerations in Web Scraping

Before scraping data, it’s essential to consider:

  • Website Terms of Service: Always check if the website permits data scraping.
  • Robots.txt File: This file indicates which parts of the site can be crawled (e.g. https://www.govinfo.gov/robots.txt)
  • Rate Limiting: Avoid overwhelming servers by limiting the frequency of requests.
  • Data Privacy: Ensure that personal or sensitive information is handled appropriately.
Code 
library(rvest)
library(polite)

# Ethical scraping session
session <- bow("https://en.wikipedia.org/wiki/List_of_presidents_of_the_United_States", 
               delay = 5, 
               user_agent = "Academic Research Bot")

# Scrape using specific table selector
presidents <- scrape(session) %>%
  html_element("table.wikitable:nth-of-type(1)") %>%  # Targets first wikitable
  html_table()

# Check results
if (length(presidents) == 0) {
  stop("Table not found - check selector or page structure")
} else {
  head(presidents)
}

3.2.3 Section 3: Accessing Data via APIs

APIs provide structured access to data and are often the preferred method for data collection.

Advantages of APIs: - Structured and reliable data. - Legal and documented access. - Often more efficient than scraping.

Example: Accessing Data from an API

Suppose we have access to an API that provides GDP data. While the specific implementation depends on the API’s structure, the general approach in R would involve:

Code 
# Load necessary library
library(httr)

# Define the API endpoint
api_url <- "https://api.example.com/gdp"

# Make a GET request
response <- GET(api_url)

# Parse the content
data <- content(response, "parsed")

# Convert to a data frame
gdp_data <- as.data.frame(data)

# View the data
head(gdp_data)

Note: Replace https://api.example.com/gdp with the actual API endpoint.

3.2.4 Section 4: AI-Powered Summarization

After collecting data, AI models like GPT can help summarize and interpret the information.

Example: Summarizing GDP Data

Assuming we have a data frame gdp_table with GDP information, we can use an AI model to summarize the top countries by GDP.

Code 
# Load necessary library
library(openai)
library(knitr)
library(ellmer)
# Prepare the prompt
top_countries <- gdp_table$country_territory[2:6]
country_list <- combine_words(top_countries)
prompt_text <- paste("Summarize the top 5 countries by GDP: ", country_list)

# Initialize the Chat object
chat_obj <- chat_openai(
  model = "gpt-4o",
  system_prompt = "You are a World economy expert."
)

chat_obj$chat(prompt_text)
As of the latest available data, the top five countries by GDP are the United 
States, China, Japan, Germany, and India. Here's a brief overview of each:

1. **United States**: The U.S. remains the world's largest economy, driven by a
diverse set of industries including technology, finance, healthcare, and 
consumer goods. The country benefits from a large and affluent consumer base, 
strong financial markets, and a robust innovation ecosystem. It also plays a 
crucial role in global trade, with significant imports and exports.

2. **China**: China ranks second in terms of GDP, underpinned by its status as 
a global manufacturing hub and a rapidly growing consumer market. Its economy 
has experienced rapid growth over the past few decades, though it faces 
challenges such as managing debt levels, shifting towards a consumption-driven 
economy, and addressing environmental concerns.

3. **Japan**: Japan is the third largest economy and is known for its advanced 
technology, automotive industry, and electronics. It has a mature economy with 
a high standard of living. Challenges include an aging population and

4. **Germany**: As Europe's largest economy, Germany is renowned for its

5. **India**: India rounds out the top five, driven by a large and youthful
