Part 1: Getting Started with Quarto

Installation & Setup

Install Quarto from quarto.org

# Verify installation
quarto --version

# Create a new project
quarto create-project my-website --type website

Project Structure

my-website/
├── _quarto.yml          # Main configuration
├── index.qmd            # Home page
├── about.qmd            # About page
├── research.qmd         # Research page
├── cv.pdf               # CV document
├── styles.css           # Custom styles
└── _site/               # Generated output (don't commit)

Part 4: Git & GitHub Setup

Initialize Git Repository

# Navigate to your project
cd my-website

# Initialize git
git init

# Add all files
git add .

# Create initial commit
git commit -m "Initial commit: Quarto website setup"

GitHub Configuration

# Create repository on GitHub first
# (https://github.com/new)

# Add remote origin
git remote add origin https://github.com/yourusername/my-website.git

# Rename branch to main
git branch -M main

# Push to GitHub
git push -u origin main

.gitignore File

# Ignore site output
_site/
.quarto/

# Ignore system files
.DS_Store
Thumbs.db

# Ignore IDE
.vscode/
.idea/

# Ignore dependencies
renv/

Part 5: Publishing to GitHub Pages

Enable GitHub Pages

  1. Go to Repository Settings
  2. Navigate to “Pages” section
  3. Select “Deploy from a branch”
  4. Choose branch: main
  5. Select folder: / (root) or /docs

Automated Publishing with Quarto

Add to _quarto.yml:

project:
  type: website
  output-dir: docs  # For GitHub Pages root

publish:
  - source: .
    netlify:
      - id: mysite
        url: https://mysite.netlify.app

Deploy Workflow

# Make changes locally
nano about.qmd

# Render the site
quarto render

# Stage changes
git add .

# Commit
git commit -m "Update about page"

# Push to GitHub
git push origin main

# Site automatically updates!

Create a Blog Section

Add to _quarto.yml:

website:
  navbar:
    left:
      - text: "Blog"
        href: blog/index.qmd

  listing:
    - id: blog-listing
      contents: "blog/*.qmd"
      sort: "date desc"
      type: default
      fields: [date, title, reading-time]

Part 7: Troubleshooting & Best Practices

Common Issues

Issue: _site folder not rendering

# Clean and rebuild
quarto clean
quarto render

Issue: Changes not pushing to GitHub

# Check git status
git status

# Verify remote
git remote -v

Best Practices

✅ Commit frequently with clear messages

✅ Keep local _site/ folder in .gitignore

✅ Use descriptive YAML configurations

✅ Test locally before pushing:

quarto preview

✅ Keep content organized in folders

✅ Use relative paths for links

❌ Don’t commit generated files
