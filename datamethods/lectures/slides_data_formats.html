EPPS 6302: Data (file) format

Karl Ho

Data import

  • Most-used R import packages
  • Conventional formats: CSV, SPSS, Stata, XML, JSON
  • Modern/columnar & array formats: Feather / Arrow IPC, Parquet, ORC, HDF5, Zarr, Avro
  • Minimal code patterns + when to use which

Import packages you’ll use most (R)

  • readr — fast, friendly CSV/TSV/fwf (read_csv())
  • data.table — ultra-fast fread() for delimited text
  • vroom — fast/lazy delimited import vroom()
  • haven — SPSS/Stata/SAS (read_sav(), read_dta())
  • jsonlite — JSON (fromJSON(), stream_in() for NDJSON)
  • xml2 — XML/HTML parsing (read_xml(), XPath helpers)
  • arrow — Parquet, Feather/Arrow IPC, CSV/JSON readers; multi-file Dataset API
  • (Also helpful: readxl for Excel; rio as a convenience wrapper)

Practical tip: prefer arrow / vroom / fread for speed; readr for rich parse diagnostics.

CSV (Comma-separated values)

What: Plain text, delimited; lingua franca for tabular data
Pros: Universal, human-readable; easy to diff/version
Cons: No schema/metadata; typing & encoding issues; large size

R patterns

readr::read_csv("file.csv")
data.table::fread("file.csv")
vroom::vroom("file.csv")
arrow::read_csv_arrow("file.csv")   # fast, Arrow-backed

When: Interchange, teaching, small–medium datasets.

SPSS (.sav/.zsav)

What: Binary format with labels & metadata
Pros: Preserves value/variable labels, missing codes
Cons: Software-specific quirks; version differences

R patterns (haven)

library(haven)
d_spss <- read_sav("survey.sav")

Stata (.dta)

What: Stata binary dataset
Pros: Labels, dates, factor support; widely used in social science
Cons: Version differences (13+ vs older), string length limits in older versions

R patterns (haven)

library(haven)
d_stata <- read_dta("study.dta")  # write_dta() to export

XML

What: Hierarchical, verbose, schema-able (XSD), namespaces
Pros: Strong validation; good for nested/meta-rich data
Cons: Verbose; XPath/namespace learning curve

R patterns

library(xml2)
doc   <- read_xml("records.xml")
nodes <- xml_find_all(doc, "//record[@status='active']")

JSON (incl. NDJSON)

What: Lightweight nested objects/arrays; web & API standard
Pros: Flexible nesting; streaming (NDJSON)
Cons: Irregular structure; rectangularizing can be tricky

R patterns

library(jsonlite)
x <- fromJSON("data.json")        # small-to-medium JSON
# For large newline-delimited JSON (NDJSON):
con <- file("data.ndjson")
big <- stream_in(con); close(con)

# Arrow reader for NDJSON:
arrow::read_json_arrow("data.ndjson")

Feather / Arrow IPC (aka Arrow file)

What: Binary columnar format for fast, cross-language data frames
Notes: Feather V2 == Arrow IPC file format (with compression)
Pros: Very fast I/O; preserves types; easy R ↔︎ Python hand-off

R patterns (arrow)

library(arrow)
write_feather(df, "data.feather")    # writes Arrow IPC (V2)
df2 <- read_feather("data.feather")  # alias: read_ipc_file()

When: Local analytics & interop between tools.

Parquet (pronounced as “parkay”)

What: Compressed columnar storage with rich schema & stats
Pros: Small on disk; column projection; widely used in data lakes
R patterns (arrow)

library(arrow)
write_parquet(df, "data.parquet")
df <- read_parquet("data.parquet", col_select = c("id","date","value"))
# Larger-than-memory:
ds <- open_dataset("s3://bucket/path/", format = "parquet")
library(dplyr)
out <- ds %>% filter(year == 2023) %>% select(id, value) %>% collect()

When: Analytics at scale; multi-file/cloud workflows.

ORC (Optimized Row Columnar)

What: Columnar format popular in Hadoop/Spark; indexes & compression
R options

  • Use Spark via sparklyr to read/write ORC
library(sparklyr)
sc <- spark_connect(master = "local")
tbl <- spark_read_orc(sc, name = "orc_tbl", path = "data.orc")

When: Interop with Hadoop/Spark; convert to Parquet for pure-R workflows.

HDF5

What: Hierarchical array format for very large/complex data
Pros: Chunked I/O; compression; random access; rich attributes
R packages

# Bioconductor
BiocManager::install("rhdf5"); library(rhdf5)
h5createFile("x.h5"); h5write(matrix(1:12,3,4), "x.h5", "grp/data")

# CRAN OO wrapper
install.packages("hdf5r"); library(hdf5r)
f <- H5File$new("x.h5", mode="a"); f[["grp/data2"]] <- as.matrix(df)
f$close_all()

When: Large matrices/arrays, bioinformatics, imaging.

Zarr (chunked, cloud-native arrays)

What: Chunked, compressed N-D arrays; cloud/object-store friendly
R options

  • Rarr for local/cloud Zarr arrays
# BiocManager::install("Rarr")
library(Rarr)
z <- ZarrArray("data.zarr")
sub <- z[1:1000, 1:1000]
  • stars (via GDAL) can read Zarr rasters when GDAL has the Zarr driver

When: Earth science, imaging, ML tensors; cloud workflows.

Avro (row-oriented, schema-first)

What: Row-oriented binary serialization with JSON schema; common in Kafka/event pipelines
R options

  • Use sparklyr (with Avro package) to read/write
library(sparklyr)
sc <- spark_connect(master = "local")
df <- spark_read_avro(sc, name = "events", path = "data.avro")
  • Or use reticulate to call Python fastavro and convert to R

When: Event streams, schema evolution; less common in base-R without Spark.

Cheat sheet: What to pick?

  • Teaching & interchange: CSV
  • Labeled survey files: SPSS/Stata via haven
  • Nested/web: JSON (NDJSON for large)
  • Analytics @ scale: Parquet (datasets on disk/cloud)
  • Fast R↔︎Python hand-off: Feather/Arrow IPC
  • Hadoop/Spark ecosystems: ORC (via sparklyr)
  • Large arrays/science: HDF5 or Zarr

Minimal install block

install.packages(c(
  "readr","data.table","vroom","haven","jsonlite","xml2","arrow","sparklyr","hdf5r"
))
# Bioconductor:

BiocManager::install(c("rhdf5","Rarr"))
