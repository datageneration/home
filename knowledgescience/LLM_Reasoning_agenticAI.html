Part I: Large Language Models

What is a Large Language Model?

An LLM is a neural network trained on massive text corpora to predict the next token in a sequence.

Core components:

  • Transformer architecture (Vaswani et al., 2017): Self-attention mechanism allows the model to weigh relationships between all tokens in a sequence
  • Pre-training: Unsupervised learning on trillions of tokens from the web, books, and code
  • Fine-tuning: Supervised instruction tuning + RLHF (Reinforcement Learning from Human Feedback) to align with human preferences
  • Scaling laws: Performance improves predictably with more parameters, data, and compute (Kaplan et al., 2020)

The Transformer Architecture

Key insight: Self-attention allows every token to attend to every other token, capturing long-range dependencies that RNNs and LSTMs struggled with.

The Self-Attention Mechanism

The core mathematical operation in transformers — Scaled Dot-Product Attention:

\[\text{Attention}(Q, K, V) = \underbrace{\text{softmax}}_{\text{normalize to probabilities}}\left(\frac{QK^T}{\underbrace{\sqrt{d_k}}_{\text{scaling factor}}}\right)V\]

Where:

  • \(Q\) = Query matrix — “what am I looking for?” Each token generates a query vector representing its information need
  • \(K\) = Key matrix — “what do I contain?” Each token generates a key vector advertising its content
  • \(V\) = Value matrix — “what information do I carry?” The actual content to be retrieved
  • \(d_k\) = Dimension of key vectors; dividing by \(\sqrt{d_k}\) prevents dot products from growing too large, which would push softmax into regions of vanishing gradients
  • \(\text{softmax}(z_i) = \frac{e^{z_i}}{\sum_j e^{z_j}}\) — converts raw attention scores into a probability distribution that sums to 1, so each token attends to a weighted mixture of all other tokens

Multi-head attention runs this operation in parallel across \(h\) heads, each learning different relationship patterns (syntax, semantics, coreference, etc.).

Training Pipeline

The pre-training objective — Cross-Entropy Loss over next-token prediction:

\[\mathcal{L}_{\text{pretrain}} = -\sum_{t=1}^{T} \underbrace{\log P(x_t | x_{<t}; \theta)}_{\text{log-likelihood of token } x_t \text{ given all preceding tokens}}\]

where \(T\) is sequence length, \(x_t\) is the token at position \(t\), and \(\theta\) are the model parameters. This simple objective, at scale, produces emergent capabilities in reasoning, translation, coding, and more.

Mixture-of-Experts (MoE)

A major architectural innovation enabling scale without proportional compute cost:

DeepSeek V3/R1: 671B total parameters, but only 37B activated per token. This sparse activation enables frontier-level performance at a fraction of the inference cost.

Part II: Reasoning Models

From Pattern Matching to Deliberate Thinking

Standard LLMs use System 1 thinking (Kahneman 2011): fast, intuitive, pattern-based.

Reasoning models introduce System 2 thinking: slow, deliberate, step-by-step.

Chain-of-Thought (CoT) Reasoning

The foundational technique for eliciting reasoning in LLMs:

Zero-shot CoT (Kojima et al., 2022): Simply appending “Let’s think step by step” to a prompt improved accuracy on GSM8K math problems from ~18% to ~79%.

Few-shot CoT (Wei et al., 2022): Providing worked examples with intermediate steps further improved performance.

Test-Time Compute Scaling

The key insight of reasoning models: spend more compute at inference time rather than only at training time.

\[\text{Performance} \propto \underbrace{f(\text{Training Compute})}_{\text{pre-training: more params, data}} + \underbrace{g(\text{Inference Compute})}_{\text{test-time: longer "thinking"}}\]

Traditional scaling focused on \(f\): more parameters, more data, more training compute.

Reasoning models exploit \(g\): the model “thinks longer” on harder problems, generating internal reasoning tokens before producing a final answer.

Controllable reasoning depth (GPT-5.2 thinking mode):

  • effort: low — fast responses, minimal deliberation
  • effort: medium — balanced reasoning
  • effort: high — extended deliberation for complex problems
  • effort: xhigh — 3–5× more internal reasoning tokens

DeepSeek R1: Open-Source Reasoning

DeepSeek R1 demonstrated a groundbreaking result: reasoning can emerge from pure reinforcement learning without human-annotated reasoning traces.

Training pipeline:

GRPO (Group Relative Policy Optimization): Samples multiple responses per input, rewards based on correctness alone. No need for human demonstrations of reasoning.

Published in Nature (Guo et al. 2025) — validating that reasoning capabilities can be incentivized purely through RL.

The GRPO Objective

DeepSeek R1’s training relies on Group Relative Policy Optimization, a variant of policy gradient methods:

\[\mathcal{L}_{\text{GRPO}} = -\mathbb{E}_{x \sim D}\left[\frac{1}{G}\sum_{i=1}^{G} \underbrace{A_i}_{\text{advantage}} \cdot \underbrace{\log \pi_\theta(y_i | x)}_{\text{log-prob of response } y_i}\right]\]

Where:

  • \(x\) = input problem sampled from dataset \(D\)
  • \(G\) = group size (number of sampled responses per input)
  • \(y_i\) = the \(i\)-th sampled response
  • \(\pi_\theta(y_i | x)\) = probability of generating response \(y_i\) under current policy \(\theta\)
  • \(A_i\) = advantage — how much better response \(y_i\) is relative to the group average:

\[A_i = \frac{r_i - \text{mean}(r_1, \dots, r_G)}{\text{std}(r_1, \dots, r_G)}\]

The reward \(r_i\) is rule-based: correct final answer \(\rightarrow\) high reward; incorrect \(\rightarrow\) low reward. No human demonstrations needed — the model discovers reasoning strategies by exploring which response patterns lead to correct answers.

The “Aha Moment”

During RL training, DeepSeek R1-Zero spontaneously developed:

  • Self-verification: Checking its own work mid-chain
  • Reflection: Recognizing when an approach is failing
  • Backtracking: Abandoning incorrect paths and trying alternatives
  • Dynamic strategy: Switching between reasoning approaches based on problem type

Key Insight

These behaviors were not programmed — they emerged as the model discovered that careful reasoning led to higher rewards. This suggests reasoning may be a natural consequence of optimizing for correctness on hard problems.

Reasoning Distillation

A critical finding: reasoning capabilities can be distilled from large models into smaller ones.

Distilled Model Base Performance
R1-Distill-Qwen-32B Qwen 32B Outperforms o1-mini
R1-Distill-Qwen-8B Qwen 8B Matches Qwen3-235B-Thinking on some tasks
R1-Distill-Llama-70B Llama 70B Competitive with full R1

Implication: Frontier-level reasoning on a single GPU. The 8B distilled model runs on 40–80GB VRAM, making reasoning research accessible to individual researchers and small labs.

Limitations of Reasoning Models

Reasoning models are powerful but not infallible:

  • Overthinking: May spend excessive compute on simple problems
  • Confident but wrong: Structured reasoning traces can make errors appear convincing
  • Compute cost: Reasoning tokens are expensive — a single o3-pro query can cost 10–100× more than a standard query
  • The “Illusion of Thinking” (Apple ML Research, 2025): Reasoning traces may not always reflect genuine logical inference
  • Repetition loops: DeepSeek R1-Zero initially suffered from endless repetition and language mixing

Critical Evaluation

A well-structured reasoning trace does not guarantee a correct answer. Always validate model outputs against ground truth, especially in research and policy contexts.

Part III: Agentic AI Systems

From Chatbots to Agents

The fundamental shift:

Feature Chatbot (LLM) Agent (Agentic AI)
Interaction Responds to prompts Pursues goals
Planning None Multi-step plans
Tool use Limited Native tool calling
Memory Context window only Short + long-term memory
Error recovery Manual re-prompting Self-correction
Autonomy None Operates for hours/days

Traditional LLMs are systems of language. Agentic AI systems are systems of behavior.

The Three Pillars of Agency

Agentic Design Patterns

Seven core patterns that form the architectural vocabulary for agent development:

  1. ReAct (Reasoning + Acting): Interleave reasoning traces with tool calls
  2. Reflection: Agent critiques its own outputs and iterates
  3. Tool Use: Native function calling to external APIs, databases, code executors
  4. Planning: Decompose complex goals into sub-tasks with dependency ordering
  5. Multi-Agent Collaboration: Specialized agents with distinct roles communicate via structured protocols
  6. Sequential Workflows: Pipeline of agent steps with conditional branching
  7. Human-in-the-Loop: Agent operates autonomously but escalates decisions to humans at defined checkpoints

The ReAct Pattern

The most widely adopted agentic pattern, combining Reasoning and Action:

Each Thought → Action → Observation cycle allows the agent to ground its reasoning in real-world data and self-correct.

Multi-Agent Systems

Complex tasks can be decomposed across specialized agents that collaborate:

Examples: MetaGPT and ChatDev assign Software PM, Architect, Developer, and Tester roles to separate agents communicating via structured documents.

Agentic Memory Types

Persistent memory is what separates a stateless chatbot from a true agent:

Memory Type Description Example
Episodic Records of past interactions and events “Last time I searched this database, I found…”
Semantic Learned facts and domain knowledge “This API requires OAuth2 authentication”
Procedural Learned strategies and procedures “For data cleaning, first check for nulls, then outliers”

These correspond to human memory systems and enable agents to improve over time rather than starting from scratch each session.

The Cost-Performance Frontier

Production agentic systems require careful model selection:

The Plan-and-Execute pattern — a frontier model plans while cheaper models execute — can reduce costs by up to 90% compared to using frontier models for everything.

Ethical and Governance Considerations

As AI systems gain autonomy, critical questions arise:

  • Accountability: When an agent makes a consequential decision, who is responsible?
  • Transparency: Private reasoning traces (o3) vs. open traces (DeepSeek R1) — should we see how AI thinks?
  • Alignment: How do we ensure long-horizon agents pursue intended goals over days of autonomous operation?
  • Access and equity: Frontier reasoning costs $15–60/M tokens — who can afford to think deeply?
  • Geopolitical: Open-source (DeepSeek, Llama) vs. closed (GPT, Claude) — implications for global AI development

Critical Question

As AI systems move from “systems you talk to” to “systems you give goals to,” how should research ethics frameworks adapt?

Summary

  1. LLMs are the foundation — transformer-based models trained on massive text to understand and generate language
  2. Reasoning models add deliberate, verifiable thinking via RL and test-time compute scaling
  3. Agentic AI combines reasoning with tool use, memory, and planning for autonomous operation
  4. The field is moving from models you talk to → models you give goals to
