Part I: Statistical Learning

What is Statistical Learning?

  • Statistical learning refers to a set of approaches for estimating the relationship between inputs and outputs

  • Given a response variable \(Y\) and \(p\) predictors \(X_1, X_2, \ldots, X_p\), we assume:

\[Y = f(X) + \epsilon\]

  • \(f\) represents the systematic information that \(X\) provides about \(Y\)
  • \(\epsilon\) is the irreducible error term

Prediction Trade-offs

  • Accuracy vs. Interpretability: More flexible models may predict better but are harder to interpret

  • Parsimony vs. All-in models: Simpler models generalize better (Occam’s razor) vs. complex models capturing more variance

  • Linear vs. Non-linear: Linear models are interpretable but constrained; non-linear models are flexible but risk overfitting

Prediction Error

The expected prediction error can be decomposed:

\[E\left[(Y - \hat{f}(X))^2\right] = \underbrace{\text{Var}(\hat{f}(X))}_{\text{Variance}} + \underbrace{[\text{Bias}(\hat{f}(X))]^2}_{\text{Bias}^2} + \underbrace{\text{Var}(\epsilon)}_{\text{Irreducible}}\]

  • Reducible error: Can be minimized by improving \(\hat{f}\)
  • Irreducible error: Intrinsic variability in \(Y\) that cannot be predicted

Parametric vs. Non-parametric Models

Parametric

  • Assume a functional form for \(f\) (e.g., linear)
  • Reduce the problem to estimating parameters
  • Risk: model misspecification
  • Examples: Linear regression, logistic regression

Non-parametric

  • Make no assumptions about the form of \(f\)
  • Require more observations to estimate \(f\)
  • Risk: overfitting
  • Examples: KNN, splines, random forests

Supervised vs. Unsupervised Learning

Supervised

  • Response variable \(Y\) is known (labeled)
  • Goal: predict \(Y\) from \(X\)
  • Regression (quantitative \(Y\))
  • Classification (qualitative \(Y\))

Unsupervised

  • No response variable \(Y\)
  • Goal: find structure in \(X\)
  • Clustering
  • Dimensionality reduction

Model Flexibility Spectrum

Low Flexibility \(\longleftrightarrow\) High Flexibility

More interpretable \(\longleftrightarrow\) More flexible (less interpretable)

Part II: Linear Regression

Linear Regression: Overview

  • A simple approach to supervised learning assuming the dependence of \(Y\) on \(X_i\) is linear

  • True regression functions are never linear!

  • Yet linear regression is very powerful and useful both conceptually and practically

Why Data Are Never Truly Linear

Real data deviate from linearity due to different types of errors:

  • Measurement error: Imprecision in recording values

  • Sampling error: Variability from observing a subset of the population

  • Multiple covariates: Omitted variables and complex interactions

Simple Linear Regression Model

Assume a model:

\[Y = \beta_0 + \beta_1 X + \epsilon\]

  • \(\beta_0\) and \(\beta_1\): unknown constants (intercept and slope), also known as coefficients or parameters
  • \(\epsilon\): the error term

Given estimates \(\hat{\beta}_0\) and \(\hat{\beta}_1\), we predict:

\[\hat{y} = \hat{\beta}_0 + \hat{\beta}_1 x\]

where \(\hat{y}\) indicates a prediction of \(Y\) on the basis of \(X = x\). The hat symbol denotes an estimated value.

Where is the \(\epsilon\)?

\[\hat{y} = \hat{\beta}_0 + \hat{\beta}_1 x\]

  • The predicted value \(\hat{y}\) does not include the error term

  • The error \(\epsilon\) represents the gap between the true \(Y\) and the estimated \(\hat{y}\)

  • We can only estimate \(f(X)\); the irreducible error remains

Estimation by Least Squares

Let \(\hat{y}_i = \hat{\beta}_0 + \hat{\beta}_1 x_i\) be the prediction for \(Y\) based on the \(i\)th value of \(X\).

Then \(e_i = y_i - \hat{y}_i\) represents the \(i\)th residual.

The Residual Sum of Squares (RSS):

\[\text{RSS} = e_1^2 + e_2^2 + \cdots + e_n^2 = \sum_{i=1}^n (y_i - \hat{\beta}_0 - \hat{\beta}_1 x_i)^2\]

The least squares approach chooses \(\hat{\beta}_0\) and \(\hat{\beta}_1\) to minimize the RSS.

Least Squares Estimators

The minimizing values are:

\[\hat{\beta}_1 = \frac{\sum_{i=1}^{n}(x_i - \bar{x})(y_i - \bar{y})}{\sum_{i=1}^{n}(x_i - \bar{x})^2}\]

\[\hat{\beta}_0 = \bar{y} - \hat{\beta}_1 \bar{x}\]

where \(\bar{y} \equiv \frac{1}{n}\sum_{i=1}^{n} y_i\) and \(\bar{x} \equiv \frac{1}{n}\sum_{i=1}^{n} x_i\) are the sample means.

Assessment: Standard Errors

  • Standard errors quantify uncertainty in estimated coefficients

  • Using SE and estimated coefficients, we compute the t-statistic for significance:

\[t = \frac{\hat{\beta}}{SE(\hat{\beta})}\]

  • A large \(|t|\) provides evidence against the null hypothesis \(H_0: \beta = 0\)

Assessment: Confidence Intervals

  • Standard errors can be used to compute confidence intervals

  • A 95% confidence interval: a range of values such that with 95% probability, it contains the true unknown parameter value

\[\hat{\beta}_1 \pm 2 \cdot SE(\hat{\beta}_1)\]

  • Confidence intervals are important for both frequentists and Bayesians to determine if a predictor is “significant”

Assessment: Model Fit

  • Residual Standard Error (RSE): Estimate of the standard deviation of \(\epsilon\)

\[\text{RSE} = \sqrt{\frac{1}{n-2}\text{RSS}}\]

  • \(R^2\) Statistic: Proportion of variance explained

\[R^2 = \frac{\text{TSS} - \text{RSS}}{\text{TSS}} = 1 - \frac{\text{RSS}}{\text{TSS}}\]

where \(\text{TSS} = \sum(y_i - \bar{y})^2\) is the Total Sum of Squares.

Multiple Linear Regression

Multiple regression extends the model to \(p\) predictors:

\[Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_p X_p + \epsilon\]

  • We interpret \(\beta_j\) as the average effect on \(Y\) of a one unit increase in \(X_j\), holding all other predictors fixed

  • Advertising example: Sales ~ TV + Radio + Newspaper

Linear Regression: Summary

Strengths:

  • Simple and highly interpretable
  • Good starting point for predictive analytics
  • Effective error management

Limitations:

  • Requires numeric, quantitative variables
  • Needs clear interpretations of coefficients
  • Other regression problems (multicollinearity, heteroscedasticity, non-linearity)
  • Next: Classification methods for qualitative responses

Part III: Classification

Classification: Overview

  • Qualitative variables take values in an unordered set \(\mathcal{C}\), such as:
    • \(\text{PartyID} \in \{\text{Democrat, Republican, Independent}\}\)
    • \(\text{Voted} \in \{\text{Yes, No}\}\)
  • Given a feature vector \(X\) and a qualitative response \(Y\) taking values in \(\mathcal{C}\), the classification task is to build a function \(C(X)\) that predicts \(Y\)

Why Not Linear Regression?

  • For a binary outcome (0/1), linear regression can still generate meaningful results

  • But for three or more categories, linear regression cannot be used given the lack of natural ordering

  • Even for binary outcomes, linear regression predictions can go out of bound (below 0 or above 1)

  • We need a model that produces outputs between 0 and 1

Probability Estimation

  • From a prediction standpoint, we estimate the probabilities that \(X\) belongs to each category in \(\mathcal{C}\)

  • Example: Will a student credit card owner default?

  • We cannot say with 100% certainty — but we can estimate the probability of default

  • Default data example: overall default rate ≈ 3%; higher balance tends to lead to default

Logistic Regression

Instead of modeling \(Y\) directly, logistic regression models the probability that \(Y\) belongs to a particular category:

\[p(X) = Pr(\text{default} = \text{Yes} \mid \text{balance})\]

  • Prediction: classify as default if \(p(X) > 0.5\)

  • Can set other thresholds (e.g., \(p(X) > 0.3\)) depending on the cost of errors

The Logistic Function

To ensure outputs between 0 and 1, we use the logistic function:

\[p(X) = \frac{e^{\beta_0 + \beta_1 X}}{1 + e^{\beta_0 + \beta_1 X}}\]

  • \(e \approx 2.71828\) (Euler’s number)
  • No matter the values of \(\beta_0\), \(\beta_1\), or \(X\), \(p(X) \in (0,1)\)

Odds and Log-Odds

The odds are defined as:

\[\frac{p(X)}{1 - p(X)} = e^{\beta_0 + \beta_1 X}\]

Taking the logarithm of both sides:

\[\log\left(\frac{p(X)}{1 - p(X)}\right) = \beta_0 + \beta_1 X\]

  • The left-hand side is called the log-odds or logit
  • The logit is linear in \(X\), but \(p(X)\) and \(X\) are not linearly related

Maximum Likelihood Estimation

Logistic regression is estimated using Maximum Likelihood:

\[\ell(\beta_0, \beta_1) = \prod_{i:y_i=1} p(x_i) \prod_{i:y_i=0} (1 - p(x_i))\]

  • We seek \(\hat{\beta}_0\) and \(\hat{\beta}_1\) that maximize the likelihood of the observed data

  • In R, we use the glm function:

glm.fit <- glm(default ~ balance, family = binomial)
summary(glm.fit)

Confounding Effect

  • Confounding: When other predictors are in the model, the student effect changes direction

  • Students tend to have higher balances than non-students → marginal default rate is higher

  • But for each level of balance, students default less than non-students

  • Multiple logistic regression reveals this by controlling for other variables

Discriminant Analysis

  • An alternative approach: model the distribution of \(X\) in each class, then use Bayes’ theorem to obtain \(Pr(Y = k \mid X = x)\)

  • Bayes classifier: predict class 1 if \(Pr(Y = 1 \mid X = x_0) > 0.5\)

  • Requires specifying:

    • \(\pi_k = Pr(Y = k)\): the prior probability for class \(k\)
    • \(f_k(x) = Pr(X = x \mid Y = k)\): the density of \(X\) in class \(k\)

Bayes’ Theorem for Classification

\[Pr(Y = k \mid X = x) = \frac{\pi_k \, f_k(x)}{\sum_{l=1}^{K} \pi_l \, f_l(x)}\]

  • \(\pi_k\): prior probability for class \(k\)
  • \(f_k(x)\): density function of \(X\) in class \(k\) (assumed Gaussian)
  • The decision boundary shifts when priors differ across classes

Linear Discriminant Analysis (LDA)

When \(p = 1\), assume Gaussian density with class-specific mean \(\mu_k\) and common variance \(\sigma^2\):

  • LDA improves prediction by accounting for the normally distributed \(X\)

  • Why not just use linear regression? LDA provides a principled probabilistic framework

LDA is preferred when:

  • Classes are well-separated (logistic regression estimates become unstable)
  • \(n\) is small and \(X\) is approximately normal in each class
  • \(K > 2\) classes

LDA vs. Logistic Regression

LDA Logistic Regression
Assumption Gaussian with common covariance No distributional assumption
When better Small \(n\), well-separated classes, \(K > 2\) Non-Gaussian distributions
Estimation Based on mean and covariance Maximum likelihood
  • LDA can outperform logistic regression when Gaussian assumptions hold
  • Logistic regression is more robust when they do not

LDA for \(p > 1\): Multivariate Case

  • Extends to multivariate Gaussian with class-specific mean vectors and common covariance matrix

  • Decision boundaries are linear functions of \(X\)

  • With three classes and \(p = 2\) predictors: ellipses contain 95% of the probability for each class

Sensitivity and Specificity

  • Sensitivity (True Positive Rate): percentage of true defaulters correctly identified
    • \(81/333 = 24.3\%\) in the default example
  • Specificity (True Negative Rate): percentage of non-defaulters correctly identified
    • \((1 - 23/9{,}667) \times 100 = 99.8\%\)
  • Trade-off: lowering the threshold increases sensitivity but decreases specificity

ROC Curve

  • The Receiver Operating Characteristics (ROC) curve displays classifier performance across all possible thresholds

  • AUC (Area Under the Curve): summarizes overall performance

    • An ideal ROC curve hugs the top left corner
    • Larger AUC = better classifier
    • AUC = 0.5 implies random guessing
    • AUC = 1.0 implies perfect classification

Part IV: Reasoning Models & Newest Trends

Beyond Traditional ML: The AI Landscape in 2026

The machine learning toolkit has expanded dramatically. While statistical learning, regression, and classification remain foundational, new paradigms have emerged:

  • Reasoning models that “think” before answering
  • Agentic AI systems that plan, act, and self-correct
  • Foundation models that learn general representations from massive data
  • Open-source revolution closing the gap with proprietary models

What Are Reasoning Models?

Traditional LLMs generate tokens left-to-right in a single pass. Reasoning models add an internal deliberation step — chain-of-thought (CoT) — before producing a final answer.

  • Model generates intermediate thinking steps (often hidden from the user)
  • Multi-stage computation during inference: the model “works through” the problem
  • Trade-off: slower and more compute-intensive, but solves harder problems

Key examples:

  • OpenAI o-series (o1, o3): step-by-step reasoning with tool use
  • DeepSeek-R1: open-source reasoning model that shocked the field in early 2025
  • Claude’s extended thinking: deliberation before responding
  • Gemini 2.5 Pro: reasoning with 1M+ token context windows

Reasoning Models: How They Work

  • Unlike standard generation: deliberation loop with self-correction
  • Inference compute scales with problem difficulty
  • The line between training and inference is blurring

Why Reasoning Matters for ML

Reasoning models connect to classical ML concepts:

  • Bias-variance trade-off: More inference compute reduces bias on hard problems but at higher cost
  • Ensemble intuition: Internal deliberation is analogous to aggregating multiple “reasoning paths”
  • Model selection: Reasoning models excel at multi-step logic, math, and code — not all tasks need them

Practical Rule of Thumb

Use standard models for routine generation tasks. Use reasoning models for complex analysis, multi-step problem-solving, and tasks requiring verification.

Agentic AI

Agentic AI systems go beyond single-turn prediction. They autonomously plan, execute, and adapt:

  • Tool use: Accessing databases, APIs, search engines, code interpreters
  • Planning: Breaking complex goals into sub-tasks
  • Reflection: Evaluating and revising their own outputs
  • Multi-agent orchestration: Specialized agents collaborating (researcher, coder, reviewer)

By 2028, Gartner projects 33% of enterprise applications will embed AI agents. The market grew from $7.6B (2025) to $10.9B (2026).

The Open-Source Revolution

The competitive landscape shifted fundamentally:

  • DeepSeek-R1 (Jan 2025): Frontier-level reasoning from a small Chinese lab with limited resources — open-source
  • GPT-OSS (2025): OpenAI released 120B and 20B open-weight models under Apache 2.0
  • Mixture-of-Experts (MoE): Architectures that activate only a fraction of parameters per query — enabling large models on small hardware

Open-weight models now match closed models on most benchmarks, giving researchers a viable self-hosted path with full data privacy and no API costs.

Multimodal AI

Models now process text, images, audio, and video together:

  • GPT-4o: Real-time text + image + audio understanding
  • Gemini 2.0/2.5: Native multimodal with long-context reasoning
  • Claude: Vision + document understanding + extended thinking

Implications for knowledge mining:

  • Analyze survey data and images/documents simultaneously
  • Extract structured data from PDFs, tables, charts
  • Bridge qualitative (text) and quantitative (numeric) analysis

AI for Science

ML is transforming scientific discovery:

  • Drug discovery: AI-designed candidates reaching mid-to-late clinical trials in 2026
  • Materials science: Generative models proposing novel compounds
  • AlphaEvolve: LLMs discovering new algorithms and mathematical proofs
  • Protein structure: AlphaFold and successors enabling structural biology at scale

The New Paradigm

AI for Science (AI4S) combines foundation models + domain knowledge + experimental validation. The goal is not to replace scientists but to accelerate discovery.

Key Takeaways

Recap: The Big Picture

  1. Statistical Learning provides the framework: prediction vs. inference, bias-variance trade-off, parametric vs. non-parametric

  2. Linear Regression is the foundation of supervised learning for quantitative \(Y\): simple, interpretable, but assumes linearity

  3. Classification extends to qualitative \(Y\): logistic regression for probability modeling, LDA for distributional approaches

  4. Model assessment matters: \(R^2\), RSE, confusion matrices, sensitivity/specificity, ROC/AUC

  5. Reasoning models & agentic AI represent the new frontier: chain-of-thought deliberation, tool use, and multi-agent systems are reshaping what ML can accomplish
